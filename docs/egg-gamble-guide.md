# 계란 도박 운영 가이드

이 문서는 `config/plugins/Skript/scripts/egg_gamble.sk` 기준의 계란 도박 시스템 운영 가이드다.

## 기능 요약

- 플레이어는 `/egg-gamble-egg <player>`로 구매한 도박용 계란만 사용할 수 있다.
- 자연에서 얻은 일반 계란은 우클릭 시 사용이 취소된다.
- 도박용 계란 10개를 던지면 라운드가 종료된다.
- 라운드 중 계란에서 나온 새끼 닭 수에 따라 정해진 보상표대로 에메랄드를 지급한다.
- 결과 메시지는 즉시 표시되고, 생성된 새끼 닭은 5초 뒤 제거된다.
- 정산 후 플레이어의 라운드 상태는 초기화된다.

## 도박용 계란 조건

스크립트는 계란 이름에 `도박용 계란`이 포함되어 있는지 검사해서 도박용 계란인지 판정한다.

| 항목 | 값 |
|------|----|
| 아이템 | `egg` |
| 이름 | `&e도박용 계란` |
| 로어 | `&7egg-gamble` |

`/egg-gamble-egg` 명령어가 위 메타를 붙인 계란 10개를 지급한다.
이름 또는 로어가 다르면 자연산 계란과 동일하게 취급되어 사용할 수 없다.

## 정산 규칙

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 라운드당 계란 수 | `10` | 10개째 도박용 계란을 던지면 정산 |
| 계란 10개 구매 가격 | `1 emerald` | `/egg-gamble-egg` 실행 시 차감 |
| 결과 유지 시간 | `5 seconds` | 정산 후 닭 제거 전까지 유지 |
| 닭 추적 반경 | `48 blocks` | 최근 투척 위치 근처에서 생성된 닭을 라운드에 귀속 |

보상표:

| 새끼 닭 수 | 지급 에메랄드 |
|------------|---------------|
| 0-2마리 | 0개 |
| 3마리 | 1개 |
| 4마리 | 2개 |
| 5마리 | 5개 |
| 6-8마리 | 10개 |
| 9-10마리 | 50개 |

예시:

```text
도박용 계란 10개 투척
새끼 닭 3마리 생성
에메랄드 1개 지급
5초 뒤 해당 새끼 닭 제거 및 상태 초기화
```

## 플레이어 동작

1. 등록된 구매 버튼을 눌러 도박용 계란 10개를 구매한다.
2. 도박용 계란을 10개 던진다.
3. 화면 액션바에서 현재 투척 수와 새끼 닭 수를 확인한다.
4. 10개째 투척 후 결과 메시지를 확인한다.
5. 정리 중에는 새 라운드를 시작할 수 없다.

일반 계란을 던지려고 하면 다음 메시지가 표시되고 투척이 취소된다.

```text
계란도박 > 자연에서 얻은 일반 계란은 사용할 수 없습니다.
```

## 운영자 명령어

`/egg-gamble-egg`는 커맨드 블럭에서 실행할 수 있다.
플레이어가 직접 실행할 경우 OP만 사용할 수 있다.

| 명령어 | 설명 |
|--------|------|
| `/egg-gamble-egg` | 최근 등록된 구매 버튼을 누른 플레이어에게 에메랄드를 차감하고 도박용 계란 10개 지급 |
| `/egg-gamble-egg <player>` | 대상 플레이어에게 에메랄드를 차감하고 도박용 계란 10개 지급 |
| `/egg-gamble-button` | 바라보는 버튼을 구매 버튼으로 등록 또는 등록 해제 |
| `/egg-gamble-reset` | 실행한 플레이어의 계란 도박 상태 초기화 |

구매 버튼 등록:

1. 구매 버튼으로 사용할 버튼을 바라본다.
2. OP 권한으로 `/egg-gamble-button`을 실행한다.
3. 버튼과 연결된 커맨드 블럭에는 `/egg-gamble-egg`를 입력한다.

커맨드 블럭 예시:

```text
/egg-gamble-egg
```

대상 플레이어가 에메랄드 `1`개 이상을 가지고 있으면 에메랄드를 차감하고 도박용 계란 10개를 지급한다.
부족하면 계란을 지급하지 않는다.

상태 초기화:

```text
/egg-gamble-reset
```

## 설정 변경

설정값은 `egg_gamble.sk` 상단 `options`에서 변경한다.

```skript
options:
    eg-prefix: &e계란도박 &8> &r
    eg-egg-name: &e도박용 계란
    eg-egg-lore: &7egg-gamble
    eg-eggs-per-round: 10
    eg-emeralds-per-bundle: 1
    eg-result-seconds: 5
    eg-track-radius: 48
```

계란 10개 구매 가격을 바꾸려면 `eg-emeralds-per-bundle` 값을 수정한다.
예를 들어 계란 10개 가격을 에메랄드 8개로 바꾸려면 다음처럼 변경한다.

```skript
eg-emeralds-per-bundle: 8
```

라운드당 계란 수를 바꾸려면 `eg-eggs-per-round` 값을 수정한다.
이 값은 `/egg-gamble-egg` 명령어가 지급하는 계란 개수이기도 하다.

보상표를 바꾸려면 `egPayout(chicks: number)` 함수의 조건을 수정한다.

## 배포 및 리로드

영속적 설정 파일은 `config/`에서 관리한다.

```text
config/plugins/Skript/scripts/egg_gamble.sk
```

현재 실행 중인 서버에 즉시 반영할 파일은 `data/`에도 있어야 한다.

```text
data/plugins/Skript/scripts/egg_gamble.sk
```

운영 중 스크립트만 다시 로드:

```bash
docker exec minecraft-paper rcon-cli "skript reload egg_gamble"
```

컨테이너 재시작 시에는 `config/`의 내용이 `/data`로 복사된다.
따라서 영속적으로 남길 수정은 반드시 `config/` 쪽 파일에 반영한다.

## 주의사항

- `/egg-gamble-egg`가 아닌 방식으로 일반 계란을 지급하면 사용할 수 없다.
- 도박용 계란 이름과 로어는 색상 코드까지 스크립트 설정과 일치해야 한다.
- 모든 버튼을 구매 버튼으로 처리하지 않는다. `/egg-gamble-button`으로 등록한 버튼만 구매 대상 플레이어를 기록한다.
- 등록된 구매 버튼을 누른 뒤 2초 안에 `/egg-gamble-egg`가 실행되어야 한다.
- 닭 추적은 최근 도박용 계란 투척 위치와 생성 위치를 기준으로 가장 가까운 플레이어에게 귀속한다.
- 여러 플레이어가 같은 장소에서 동시에 던지면 가까운 최근 투척자에게 닭이 귀속될 수 있다.
- 서버가 결과 유지 5초 안에 종료되면 이미 생성된 닭이 자동 정리되지 않을 수 있다.
